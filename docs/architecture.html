Executive Summary

This Terraform module deploys a complete AWS Retrieval-Augmented Generation (RAG) pipeline using Amazon Bedrock Knowledge Bases, OpenSearch Serverless for vector storage, S3 for document ingestion, Lambda for triggering ingestion jobs, Step Functions for orchestration, and EventBridge for scheduling. It enables organisations to build AI-powered knowledge retrieval systems with configurable chunking strategies, embedding models, and automated ingestion workflows.

Data Flow

  1. Documents are uploaded to the S3 data source bucket under the configured prefix (default: documents/)
  2. EventBridge Schedule triggers the Step Functions state machine on the configured cadence
  3. Step Functions invokes the Lambda ingestion function with KB and data source IDs
  4. Lambda calls bedrock-agent:startIngestionJob to begin processing
  5. Bedrock reads documents from S3, applies the chunking strategy (FIXED_SIZE: 300 tokens, 20% overlap)
  6. Embedding Model (amazon.titan-embed-text-v2:0) generates vector embeddings for each chunk
  7. OpenSearch Serverless indexes the vectors into the configured index with text and metadata fields
  8. Step Functions polls ingestion status every 30 seconds until COMPLETE or FAILED
  9. Query path: Users call the Bedrock Retrieve API, which searches OpenSearch and returns relevant chunks
  10. Foundation Model uses retrieved chunks as context to generate grounded responses

Security Controls

S3 Encryption

KMS server-side encryption with bucket key enabled. Public access fully blocked on all four settings.

OpenSearch Policies

Three-layer security: encryption policy, network policy, and data access policy with principal-based control.

IAM Least Privilege

Dedicated IAM roles for Bedrock KB, Lambda, and Step Functions. Each scoped to minimum required permissions.

Lambda Concurrency

Reserved concurrent executions (default: 5) prevent runaway invocations. 30-day log retention.

S3 Versioning

Bucket versioning enabled to protect against accidental overwrites and deletions of source documents.

Account Scoping

OpenSearch data access policy scoped to specific account IDs via allowed_account_ids variable.

Production Readiness Checklist

  • Configure KMS CMK for S3 bucket encryption (instead of AWS-managed key)
  • Enable OpenSearch standby replicas for high availability
  • Set appropriate Lambda memory and timeout for document volume
  • Configure ingestion schedule matching document update frequency
  • Restrict allowed_account_ids to known accounts only
  • Tune chunking strategy and parameters for your document type
  • Set up CloudWatch alarms on Lambda errors and Step Function failures
  • Enable VPC configuration for Lambda if network isolation is required
  • Review and customise embedding model selection for your language/domain
  • Tag all resources for cost allocation and compliance tracking
  • Test ingestion pipeline with representative document corpus before go-live
  • Configure S3 lifecycle policies for old document versions

Deployment

module "rag_pipeline" {
  source = "path/to/terraform-aws-rag-pipeline"

  name = "my-rag"

  # Embedding
  embedding_model_id = "amazon.titan-embed-text-v2:0"

  # Chunking
  chunking_strategy        = "FIXED_SIZE"
  chunk_max_tokens         = 300
  chunk_overlap_percentage = 20

  # Data source
  data_source_prefix = "documents/"

  # Automation
  ingestion_schedule = "rate(1 day)"

  tags = {
    Environment = "production"
    Project     = "knowledge-base"
  }
}

terraform init
terraform plan
terraform apply

# Upload documents to trigger ingestion
aws s3 cp ./documents/ s3://BUCKET_NAME/documents/ --recursive
